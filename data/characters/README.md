# data/characters/

This folder holds **project-specific** character data — outputs generated by the Phase 0.5 agent that are scoped to *this* project's lore and scenario.

## Structure

```
data/characters/
  [Name]/
    personality_signature.json     ← Phase 0.5-B (agent-generated, user-editable)
    examples/
      emotional_states/            ← Phase 0.5-B image outputs
        happy.png
        sad.png
        angry.png
        surprised.png
        laughing.png
        crying.png
```

## Notes

- Each character folder must match the character name **exactly** as it appears in `data/lore.json` and the scenario files (e.g. `data/scenario_scenes.json`).
- `personality_signature.json` is project-specific: the same person may behave differently in a different project's lore.
- The canonical, cross-project character library lives in `global_characters/` at the project root.
