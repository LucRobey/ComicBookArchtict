# global_characters/

This folder is the **canonical, permanent character library**. It lives outside any individual project and is shared across all stories that feature these characters.

## Structure

```
global_characters/
  [Name]/
    originals/                     ← User provides: photos, reference images
    presentation.json              ← User provides/edits: full biography
    general_mood.md                ← User provides: life context for this trip / story period
    canonical_visual.md            ← GENERATED (Phase 0.5-A) — agent writes based on originals + bio
    examples/
      turnarounds/                 ← GENERATED (Phase 0.5-A)
        front.png
        3q.png
        profile.png
        back.png
        expressions.png
```

## Rules

1. **Never delete** anything in `originals/` — these are the user's source-of-truth references.
2. `canonical_visual.md` is regenerated by the `05_visual_signature.md` pipeline. Flag it with `[REWRITE_VISUAL]` if you want the agent to update it.
3. `general_mood.md` is user-written context about the character's headspace during the current story period (not project-specific).
4. Project-specific emotional states and personality signatures go in `[project]/data/characters/[Name]/`.

## How to add a new character

1. Create `global_characters/[Name]/originals/` and drop in your reference images.
2. Write `global_characters/[Name]/presentation.json` — full biography.
3. Write `global_characters/[Name]/general_mood.md` — current life context.
4. Run the Phase 0.5-A agent pipeline (`05_visual_signature.md`) to generate `canonical_visual.md` and turnaround images.
