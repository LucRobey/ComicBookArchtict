# Pipeline 07 — Mood Simulation

**Phase:** 0.5-D  
**Output target:** `data/character_moods.json`  
**Triggered by:** User request to simulate character emotional arcs across the full scenario

---

## Purpose

This pipeline reads the full scenario and each character's personality signature, then simulates — scene by scene — what each character is experiencing emotionally. The result is `character_moods.json`, which becomes an upstream dependency for Phase 1 (character intros) and Phase 3 (dialogue writing).

Think of this as the agent "playing" every character simultaneously through every scene.

---

## Input Requirements

Before running, confirm:

- `data/scenario_scenes.json` — full scene list (required)
- `data/final_lore.json` — cast list and world context (required)
- `data/personality_signature.json` — for **every character in the cast** (required)
- `global_characters/[Name]/general_mood.md` — for each character (recommended)

If `data/personality_signature.json` is missing, run Pipeline 06 first.

---

## Step 1 — Build the Character Map

Extract from `final_lore.json`:
- Full cast list (names only)
- Relationships between characters

For each character, read their entry in `data/personality_signature.json`. Note:
- Core wound
- Mask vs. core gap
- Relationship dynamics with each other cast member
- Scene-level notes

---

## Step 2 — Simulate Scene by Scene

For each scene in `scenario_scenes.json`, for each character **present in that scene**:

Answer these questions:

1. **Dominant emotion:** What single emotion is most present for this character at the START of this scene?  
   Use only these 12 named emotions: `joyful`, `content`, `anxious`, `sad`, `angry`, `ashamed`, `performing`, `numb`, `tender`, `determined`, `overwhelmed`, `resigned`

2. **Feels (internal):** What is this character privately experiencing? (1-2 sentences — not what they show, but what they feel)

3. **Shows (external):** How does their mask express this emotion outward? (1-2 sentences — behaviour, body language, speech pattern)

4. **Tension with:** If there is a specific tension or unresolved charge between this character and one other character in the scene, name that character. Otherwise `null`.

5. **Agenda (what they want):** What does this character actually want to achieve in this scene/interaction, and what do they expect out of it? (1-2 sentences)

6. **Subtext (implied truth):** What is unspoken or implied under their actions and lines? What are the underlying currents? (1-2 sentences)

7. **Secret (what they hide):** What specific piece of information, feeling, or truth are they actively keeping from the other person in this specific scene? (1-2 sentences)

8. **Status Dynamic (power dynamic):** What is their status/power posture compared to others in this scene? Does it shift? (e.g. low status seeking control, high status collapsing, etc.)

9. **Tactics (verbal/behavioral tactics):** What specific conversational tactics do they use to get what they want? (e.g. deflection, fast-talking, silence, guilt-tripping, flattery)

10. **Stakes (cost of failure):** What is at risk for them? What happens if they fail to achieve their agenda in this scene? (1-2 sentences)

---

## Step 3 — Write `character_moods.json`

Use this exact schema:

```json
{
  "_schema_version": "1.0",
  "_description": "Per-scene mood data. Generated by Pipeline 07. Editable in Characters Hub → Mood Arc tab.",
  "characters": ["Name1", "Name2", ...],
  "scenes": [
    {
      "scene_id": 1,
      "title": "Scene title from scenario_scenes.json",
      "moods": {
        "Name1": {
          "dominant_emotion": "anxious",
          "feels": "Internal experience — private truth",
          "shows": "External behaviour — the mask",
          "tension_with": "Name2 or null",
          "agenda": "What the character wants in this scene",
          "subtext": "What they actually mean/imply under the lines",
          "secret": "What they are actively hiding",
          "status_dynamic": "Perceived power stance or shift",
          "tactics": "Verbal/conversational methods used",
          "scene_stakes": "What is at risk if they fail"
        },
        "Name2": {
          "dominant_emotion": "performing",
          "feels": "...",
          "shows": "...",
          "tension_with": null,
          "agenda": "...",
          "subtext": "...",
          "secret": "...",
          "status_dynamic": "...",
          "tactics": "...",
          "scene_stakes": "..."
        }
      }
    }
  ]
}
```

**Rules:**
- Only include characters who appear in `characters_present` for that scene
- `dominant_emotion` must be one of the 12 named values above
- `tension_with` must be a character name or `null` — never a description
- Do not invent scenes not in `scenario_scenes.json`

---

## Step 4 — Cross-Check Pass

After writing the full JSON:

1. Read through the mood arc for each character individually — does their emotional journey make sense as a story?
2. Check that the internal/external gap (feels vs. shows) is consistent with their personality signature
3. Check that tension markers are reciprocal or intentionally one-sided (note if one-sided)

Adjust any inconsistencies and rewrite the affected scene entries.

---

## Step 5 — Output Summary

```
✓ character_moods.json generated
  Characters: [list]
  Scenes simulated: [N]
  Tension pairs identified: [list any named tensions]
```

---

## Flag Handling

When user edits a mood card in the app (Mood Arc sub-tab), the app saves directly to `character_moods.json` via `/api/save`.

If the agent is asked to **re-simulate** after major scenario changes:
1. Read the current `character_moods.json` to check for any user-edited entries
2. Mark user-edited entries as `"_user_edited": true` (check git diff or ask user)
3. Re-run the simulation but **preserve** `_user_edited` entries unless explicitly told to overwrite
